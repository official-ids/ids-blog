# Автоматизация рутины: GitHub Actions для личного блога 🤖

Зачем поднимать сервер, базу данных и CMS, если у тебя уже есть GitHub?

## Идея

Каждый раз, когда я хотел что-то написать, у меня были варианты:

- 📝 Medium / Habr — красиво, но чужая платформа
- 🖥️ Свой сайт — дорого и долго поддерживать
- 📂 GitHub Gist — слишком просто, без ленты

А тут подумал — а почему бы не сделать **ленту прямо в README**?

## Как это работает

Всё строится на трёх китах:

1. **Папка `posts/`** — сюда кидаю `.md` файлы
2. **Python-скрипт** — парсит frontmatter и собирает ленту
3. **GitHub Action** — запускает скрипт при каждом пуше

```yaml
on:
  push:
    paths:
      - 'posts/**'
```

Пуш → Action → парсинг → новый README → готово!

## Результат

Теперь процесс написания поста выглядит так:

```bash
git add posts/моя-статья.md
git commit -m "📝 Новая статья"
git push
```

Через минуту на главной красивая карточка с превью, тегами и ссылкой. Без единого сервера.
