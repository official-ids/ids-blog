## 1. Форк и настройка

### 🎯 Быстрый старт

```bash
# 1. Форкните этот репозиторий (кнопка "Fork" вверху)

# 2. Клонируйте свой форк
git clone https://github.com/ВАШ_НИК/имя-репо.git
cd имя-репо

# 3. Отредактируйте config.json под себя
#    - Измените title, description, author
#    - Настройте цвета тегов
#    - Включите/выключите фичи
```

### ⚙️ Конфигурация (`config.json`)

```json
{
  "blog": {
    "title": "🎵 Название вашего блога",
    "description": "Краткое описание",
    "author": "Ваше имя",
    "posts_per_page": 10,
    "show_tags": true,
    "show_preview": true
  },
  "theme": {
    "tag_colors": {
      "ваш-тег": "цвет_в_hex"
    }
  },
  "features": {
    "tag_cloud": true,
    "last_updated_badge": true,
    "random_post_link": true,
    "post_count_badge": true
  }
}
```

## 2. Добавление первого поста

### 📝 Создание поста

1. Создайте файл в папке `posts/`:
   ```
   posts/2024-04-07_название-поста.md
   ```

2. Формат файла (обязательно с frontmatter):

```markdown
---
title: "Заголовок поста"
date: 2024-04-07
tags: [тег1, тег2]
preview: "Краткое описание для ленты (1-2 предложения)"
---

Основной текст поста...

Можно использовать **markdown**, код:

```python
print("Привет, мир!")
```

И списки, и картинки!
```

### 🚀 Публикация

```bash
# Добавьте файл
git add posts/2024-04-07_название-поста.md

# Закоммитьте
git commit -m "📝 Новый пост: Название"

# Запушьте
git push origin main
```

Через 1-2 минуты GitHub Action автоматически обновит `README.md` с новым постом! ✨

## 3. Локальная отладка

### 🖥️ Запуск генератора локально

```bash
# Убедитесь, что Python 3.7+ установлен
python --version

# Запустите генератор
python scripts/generate_feed.py

# Проверьте результат
cat README.md
cat feed.json
```

### 🐛 Решение проблем

| Ошибка | Решение |
|--------|---------|
| `Файл не имеет frontmatter` | Добавьте `---` в начало файла |
| `Отсутствует обязательное поле` | Проверьте title, date, preview |
| `Некорректный frontmatter` | Убедитесь, что есть закрывающий `---` |

## 4. Кастомизация визуала

### 🎨 Настройка цветов

В `config.json` измените цвета тегов:

```json
"theme": {
  "tag_colors": {
    "разработка": "6e40c9",  // Фиолетовый
    "музыка": "e36209",       // Оранжевый
    "идеи": "bf3989"          // Розовый
  }
}
```

Цвета берутся из [shields.io](https://shields.io/) - можно использовать любые hex-цвета.

### 🏷️ Добавление новых тегов

Просто используйте новый тег в посте - он автоматически появится в облаке тегов с дефолтным цветом. Чтобы задать свой цвет - добавьте в `config.json`.

## 5. Ограничения Гитхаба

### ⚠️ Что нужно знать

| Ограничение | Обход |
|-------------|-------|
| GitHub Actions: 2000 мин/мес (бесплатно) | Хватит для блога с запасом |
| README не поддерживает `<style>` | Используем бейджи shields.io |
| Нет серверной части | Всё генерируется в статические файлы |
| CORS для внешних ресурсов | Используем только HTTPS CDN |

### ✅ Что работает отлично

- Автоматическое обновление через Actions
- Красивые бейджи через shields.io
- Полная бесплатность (публичные репо)
- Мгновенная доступность через GitHub Pages

## 💡 Советы

1. **Именуйте файлы**: `YYYY-MM-DD_название.md` для правильной сортировки
